The Content Optimization API returns actionable recommendations for improving how AI assistants perceive and cite specific pages on your site. Each recommendation identifies a gap between how AI models currently represent your content and how they should, and suggests concrete changes — such as structural improvements, missing topics, or clarity enhancements — that you can make to close that gap.

Use the list endpoint to retrieve all recommendations for an asset, and the retrieve endpoint to fetch the full details of a single optimization record.

## List Optimizations for an Asset

Retrieve all content optimization recommendations for a tracked asset (URL).

### Parameters

<ParamField path="assetID" type="string" required>
  The Profound asset ID for the URL you want optimization recommendations for.
</ParamField>

<ParamField query="category_id" type="string">
  Filter recommendations to those associated with a specific prompt category. Useful for prioritizing optimizations within a single vertical.
</ParamField>

<ParamField query="status" type="string">
  Filter by recommendation status. Accepted values: `"open"`, `"in_progress"`, `"resolved"`.
</ParamField>

<ParamField query="priority" type="string">
  Filter by priority level: `"high"`, `"medium"`, or `"low"`.
</ParamField>

<ParamField query="limit" type="integer">
  Maximum number of results to return per page.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor returned by a previous list response. Pass this value to retrieve the next page of results.
</ParamField>

<Note>
  An asset must be tracked in Profound before optimization recommendations are generated. Add URLs through the Profound dashboard or via the assets API.
</Note>

### Code Examples

<CodeGroup>

```bash curl
curl --request GET \
  --url "https://api.profound.ai/v1/content/optimization/asset_abc123" \
  --header "Authorization: Bearer $PROFOUND_API_KEY"
```

```typescript TypeScript
import { ProfoundClient } from "@profoundai/client";

const client = new ProfoundClient({ apiKey: process.env.PROFOUND_API_KEY });

const recommendations = await client.content.optimization.list("asset_abc123", {
  category_id: "cat_xyz789",
});

console.log(recommendations);
```

```python Python
import os
from profound import ProfoundClient

client = ProfoundClient(api_key=os.environ["PROFOUND_API_KEY"])

recommendations = client.content.optimization.list(
    "asset_abc123",
    category_id="cat_xyz789",
)

print(recommendations)
```

</CodeGroup>

## Retrieve a Single Optimization

Fetch the full details of a specific content optimization recommendation by its ID.

### Parameters

<ParamField path="contentID" type="string" required>
  The unique identifier of the optimization recommendation to retrieve.
</ParamField>

<ParamField query="asset_id" type="string" required>
  The Profound asset ID associated with this optimization record.
</ParamField>

### Code Examples

<CodeGroup>

```bash curl
curl --request GET \
  --url "https://api.profound.ai/v1/content/optimization/rec_def456?asset_id=asset_abc123" \
  --header "Authorization: Bearer $PROFOUND_API_KEY"
```

```typescript TypeScript
import { ProfoundClient } from "@profoundai/client";

const client = new ProfoundClient({ apiKey: process.env.PROFOUND_API_KEY });

const recommendation = await client.content.optimization.retrieve(
  "rec_def456",
  {
    asset_id: "asset_abc123",
  }
);

console.log(recommendation);
```

```python Python
import os
from profound import ProfoundClient

client = ProfoundClient(api_key=os.environ["PROFOUND_API_KEY"])

recommendation = client.content.optimization.retrieve(
    "rec_def456",
    asset_id="asset_abc123",
)

print(recommendation)
```

</CodeGroup>

<Tip>
  Sort recommendations by `priority` and tackle `"high"` priority items first — these typically correspond to the largest measurable gains in AI visibility score.
</Tip>

## Response Fields

### List Response

<ResponseField name="data" type="array">
  List of content optimization recommendation records for the asset.

  <Expandable title="data[]">
    <ResponseField name="id" type="string">
      Unique identifier for this optimization recommendation.
    </ResponseField>

    <ResponseField name="asset_id" type="string">
      The Profound asset ID this recommendation applies to.
    </ResponseField>

    <ResponseField name="url" type="string">
      The URL of the page this recommendation is for.
    </ResponseField>

    <ResponseField name="category_id" type="string">
      The prompt category this recommendation is scoped to.
    </ResponseField>

    <ResponseField name="priority" type="string">
      Priority level: `"high"`, `"medium"`, or `"low"`.
    </ResponseField>

    <ResponseField name="status" type="string">
      Current status: `"open"`, `"in_progress"`, or `"resolved"`.
    </ResponseField>

    <ResponseField name="recommendation_type" type="string">
      The category of improvement suggested (e.g., `"add_missing_topic"`, `"improve_structure"`, `"increase_clarity"`).
    </ResponseField>

    <ResponseField name="summary" type="string">
      A brief human-readable summary of the recommendation.
    </ResponseField>

    <ResponseField name="estimated_visibility_impact" type="number">
      Projected improvement in visibility score if this recommendation is implemented, expressed as a decimal.
    </ResponseField>

    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp of when this recommendation was generated.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="meta" type="object">
  Pagination metadata.

  <Expandable title="meta">
    <ResponseField name="total_records" type="integer">
      Total number of optimization recommendations for this asset.
    </ResponseField>

    <ResponseField name="cursor" type="string">
      Cursor to pass as the `cursor` parameter to retrieve the next page of results. Absent when there are no further pages.
    </ResponseField>
  </Expandable>
</ResponseField>

### Retrieve Response

<ResponseField name="id" type="string">
  Unique identifier for the recommendation.
</ResponseField>

<ResponseField name="asset_id" type="string">
  The Profound asset ID this recommendation applies to.
</ResponseField>

<ResponseField name="url" type="string">
  The URL of the page.
</ResponseField>

<ResponseField name="priority" type="string">
  Priority level: `"high"`, `"medium"`, or `"low"`.
</ResponseField>

<ResponseField name="status" type="string">
  Current status: `"open"`, `"in_progress"`, or `"resolved"`.
</ResponseField>

<ResponseField name="recommendation_type" type="string">
  The category of improvement suggested.
</ResponseField>

<ResponseField name="summary" type="string">
  A brief summary of the recommendation.
</ResponseField>

<ResponseField name="details" type="object">
  Full details of the recommendation.

  <Expandable title="details">
    <ResponseField name="problem_description" type="string">
      A detailed explanation of the gap or issue identified in the content.
    </ResponseField>

    <ResponseField name="suggested_action" type="string">
      Specific, actionable guidance for how to address the issue.
    </ResponseField>

    <ResponseField name="affected_prompts" type="array">
      List of prompt IDs where this gap was observed.
    </ResponseField>

    <ResponseField name="example_ai_response" type="string">
      An example AI-generated answer that illustrates the problem.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="estimated_visibility_impact" type="number">
  Projected improvement in visibility score if implemented.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the recommendation was generated.
</ResponseField>
