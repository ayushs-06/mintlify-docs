Folders give you a lightweight way to organize documents within a Profound knowledge base. You might create separate folders for different product lines, content teams, or document types — and then scope knowledge base searches to a specific folder using the `folder_id` parameter. Folders do not affect indexing; they serve as a logical grouping layer on top of your document collection.

## Create a Folder

Add a new folder to an existing knowledge base.

### Parameters

<ParamField path="knowledgeBaseID" type="string" required>
  The unique identifier of the knowledge base to create the folder in.
</ParamField>

<ParamField body="name" type="string" required>
  The display name for the new folder (e.g., `"Product Documentation"`, `"Support Articles"`).
</ParamField>

### Code Examples

<CodeGroup>

```bash curl
curl --request POST \
  --url "https://api.profound.ai/v1/knowledge-bases/kb_abc123/folders" \
  --header "Authorization: Bearer $PROFOUND_API_KEY" \
  --header "Content-Type: application/json" \
  --data '{"name": "Product Documentation"}'
```

```typescript TypeScript
import { ProfoundClient } from "@profoundai/client";

const client = new ProfoundClient({ apiKey: process.env.PROFOUND_API_KEY });

const folder = await client.knowledgeBases.folders.create("kb_abc123", {
  name: "Product Documentation",
});

console.log(folder.id, folder.name);
```

```python Python
import os
from profound import ProfoundClient

client = ProfoundClient(api_key=os.environ["PROFOUND_API_KEY"])

folder = client.knowledge_bases.folders.create(
    "kb_abc123",
    name="Product Documentation",
)

print(folder.id, folder.name)
```

</CodeGroup>

## Delete a Folder

Remove a folder from a knowledge base. Documents inside the folder are not deleted — they become unfoldered and remain searchable in the knowledge base.

### Parameters

<ParamField path="knowledgeBaseID" type="string" required>
  The unique identifier of the knowledge base containing the folder.
</ParamField>

<ParamField body="folder_id" type="string" required>
  The unique identifier of the folder to delete.
</ParamField>

<Note>
  Deleting a folder does not delete its documents. Any documents assigned to the deleted folder will remain in the knowledge base and continue to appear in searches — they will simply no longer be associated with a folder.
</Note>

### Code Examples

<CodeGroup>

```bash curl
curl --request DELETE \
  --url "https://api.profound.ai/v1/knowledge-bases/kb_abc123/folders" \
  --header "Authorization: Bearer $PROFOUND_API_KEY" \
  --header "Content-Type: application/json" \
  --data '{"folder_id": "folder_def456"}'
```

```typescript TypeScript
import { ProfoundClient } from "@profoundai/client";

const client = new ProfoundClient({ apiKey: process.env.PROFOUND_API_KEY });

await client.knowledgeBases.folders.delete("kb_abc123", {
  folder_id: "folder_def456",
});

console.log("Folder deleted.");
```

```python Python
import os
from profound import ProfoundClient

client = ProfoundClient(api_key=os.environ["PROFOUND_API_KEY"])

client.knowledge_bases.folders.delete(
    "kb_abc123",
    folder_id="folder_def456",
)

print("Folder deleted.")
```

</CodeGroup>

## Typical Workflow

Here is a common pattern for creating a folder and adding documents to it in one flow:

<CodeGroup>

```typescript TypeScript
import { ProfoundClient } from "@profoundai/client";

const client = new ProfoundClient({ apiKey: process.env.PROFOUND_API_KEY });

// 1. Create a folder
const folder = await client.knowledgeBases.folders.create("kb_abc123", {
  name: "Q1 2025 Reports",
});

// 2. Add documents into that folder
await client.knowledgeBases.documents.create("kb_abc123", {
  content: "Q1 2025 visibility summary: brand mentions up 18% YoY...",
  filename: "q1-2025-visibility.md",
  folder_id: folder.id,
});

console.log(`Document added to folder: ${folder.name}`);
```

```python Python
import os
from profound import ProfoundClient

client = ProfoundClient(api_key=os.environ["PROFOUND_API_KEY"])

# 1. Create a folder
folder = client.knowledge_bases.folders.create(
    "kb_abc123",
    name="Q1 2025 Reports",
)

# 2. Add a document into that folder
client.knowledge_bases.documents.create(
    "kb_abc123",
    content="Q1 2025 visibility summary: brand mentions up 18% YoY...",
    filename="q1-2025-visibility.md",
    folder_id=folder.id,
)

print(f"Document added to folder: {folder.name}")
```

</CodeGroup>

## Response Fields

### Create Response

<ResponseField name="id" type="string">
  The unique identifier of the newly created folder.
</ResponseField>

<ResponseField name="knowledge_base_id" type="string">
  The ID of the knowledge base this folder belongs to.
</ResponseField>

<ResponseField name="name" type="string">
  The display name of the folder.
</ResponseField>

<ResponseField name="document_count" type="integer">
  Number of documents currently assigned to this folder.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the folder was created.
</ResponseField>

### Delete Response

A successful delete returns an HTTP `204 No Content` response with an empty body.
